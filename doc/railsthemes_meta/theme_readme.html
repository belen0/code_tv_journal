Theme Usage & Style Guide

About Theme Meta

Features

  • Responsive design
  • Easily customized colors (via. Sass)
  • Matching jQuery UI theme (using the same color-changing Sass)

Theme Layouts

This theme comes with a flexible application layout that allows for a left or right sidebar as well as no sidebars at all.

Sample Markup

After installing the theme, you will find that there are several dozen example haml (or erb) views available to show you exactly how to achieve any sort of page layout.

Grid System

This theme is based on the Zurb Foundation grid system.

You can view full documentation of the grid system at http://foundation.zurb.com/.

Browser Support

A browser support report (and example screenshots) can be found at https://railsthemes.com/themes/meta.

Gem Support

In addition to all of the standard styling/markup support, this gem includes special stylesheets for:

  • formtastic
  • jquery_ui
  • kaminari
  • rails standard form generators
  • simple form
  • will_paginate

Stylesheets are automatically included by the installer based on the contents of your Gemfile.

Getting Started

0. Check out the existing improvments to your app

All of your standard page elements (headers, tables, forms, etc.) should look pretty awesome already!

1. Check out the sample views

We include a variety of sample views to help you get started.

You can view the full list of sample views at http://localhost:3000/railsthemes

2. Swap in your logo and app name

In the app/views/layouts/_header_navigation.html.haml partial, swap in your application name and contact information.

In the app/views/layouts/_footer_navigation.html.haml partial, swap in your application name (or logo) and update the navigation to match your app.

In the included layouts (landing and application) you may want to edit the title tag to include your application name.

3. Take advantage of theme-specific features

If you set content_for :page_title, "My Page Title in any your views, the page title will automatically be reflected on the page.

If you set content_for :title, "My Title in any your views, the page title will automatically be included in the title tag of the page.

Your theme also includes specific partials and grid options that you might want to try out. You can view demos of them in the sample views and in the official theme demo applications.

About RailsThemes

All of our themes take advantage of the Rails asset pipeline. As such, you'll find our generated files in the following locations:

Directory Structure

  • app
    • assets
      • images
      • javascripts
      • stylesheets
    • controllers
      • railsthemes (example only)
    • views
      • layouts
        • application
        • landing
        • _flash
        • _footer_navigation
        • _header_navigation
      • railsthemes
        • inner (example of general layout)
        • landing (example of landing layout)
        • jquery_ui (example of all jquery UI elements)
  • public
    • favicon.ico

Naming Conventions

Most theme files are either namespaced (e.g. railsthemes_themename_filename.ext) or in a railsthemes specific folder (e.g. railsthemes_themename/)

The exceptions to this are images and standard css/js assets from elsewhere (e.g. html5shiv).

Rails Version Support

Our themes are designed for Rails 3.2 and up.
